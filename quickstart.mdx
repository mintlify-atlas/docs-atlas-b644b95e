# Quick Start Guide

This guide will walk you through creating a working dependent dropdown example. We'll build a three-level dropdown chain where each selection controls the next dropdown's visibility.

<Info>
  Make sure you've [installed ngx-depend-dropdown](/installation) before starting this guide.
</Info>

## What We'll Build

We're creating a user selection flow:
1. **First dropdown**: Select a user (loaded from API)
2. **Second dropdown**: Appears after first selection (dependent on user)
3. **Third dropdown**: Appears after second selection with validation

<Steps>

<Step title="Import Required Dependencies">

In your component TypeScript file, import the necessary modules and types:

```typescript app.component.ts
import { Component, OnInit } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { map } from 'rxjs';
import { NgxDependDropdownService } from 'ngx-depend-dropdown';
import { DropdownConfig, DropdownData } from 'NgxDependDropdown';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent implements OnInit {
  data: DropdownData = [];
  config: DropdownConfig = {};

  constructor(
    private http: HttpClient,
    private dropdownService: NgxDependDropdownService
  ) {}

  ngOnInit(): void {
    // We'll add configuration in the next step
  }
}
```

<Note>
  The `NgxDependDropdownService` is injected to access selected values and manage dropdown state.
</Note>

</Step>

<Step title="Configure Dropdown Behavior">

Set up the global configuration for your dropdowns:

```typescript app.component.ts
ngOnInit(): void {
  this.config = {
    unavailableType: 'disabled',
    containerClass: ['dropdown-container', 'row'],
    nodeClass: ['form-select', 'mb-3']
  };
  this.setDropdownData();
}
```

Configuration options:
- `unavailableType`: How to handle unavailable dropdowns ('disabled' or 'invisible')
- `containerClass`: CSS classes for the container wrapper
- `nodeClass`: CSS classes applied to each dropdown element

</Step>

<Step title="Define Dropdown Data">

Create the dropdown configuration with dependency rules:

```typescript app.component.ts
setDropdownData() {
  this.data = [
    {
      name: 'user-selector',
      visibility: () => true,  // Always visible
      selectedValue: null,
      placeholder: 'Select a user',
      values: [],
      setValues: () => {
        // Load data from API using Observable
        return this.http
          .get('https://jsonplaceholder.typicode.com/users')
          .pipe(
            map((data: any) => {
              return data.map((item: any) => ({
                name: item.name,
                id: item.id
              }));
            })
          );
      }
    },
    {
      name: 'dependent-selector',
      visibility: () => {
        // Only visible when user is selected
        return !!this.dropdownService.getNodeValue('user-selector');
      },
      dependent: 'user-selector',  // Depends on first dropdown
      selectedValue: null,
      placeholder: 'Select an option',
      values: [],
      setValues: () => {
        return this.http
          .get('https://jsonplaceholder.typicode.com/users')
          .pipe(
            map((data: any) => {
              return data.map((item: any) => ({
                name: item.email,
                id: item.id
              }));
            })
          );
      }
    },
    {
      name: 'final-selector',
      visibility: () => {
        // Only visible when second dropdown has value
        return !!this.dropdownService.getNodeValue('dependent-selector');
      },
      dependent: 'dependent-selector',  // Depends on second dropdown
      selectedValue: null,
      placeholder: 'Select final option',
      label: 'Final Selection',
      required: true,  // This field is required
      errorMessage: 'Please select a final option',
      values: [],
      setValues: () => {
        // Return static array instead of Observable
        return [
          { name: 'Option 1', id: 1 },
          { name: 'Option 2', id: 2 },
          { name: 'Option 3', id: 3 }
        ];
      }
    }
  ];
}
```

<Tip>
  You can use either Observables (for API calls) or static arrays in `setValues()`. The library handles both automatically.
</Tip>

</Step>

<Step title="Handle Selection Changes">

Add an event handler to respond to dropdown changes:

```typescript app.component.ts
change(event: any) {
  console.log('Changed dropdown:', event.changedDropdown.name);
  console.log('Selected value:', event.changedDropdown.selectedValue);
  console.log('All dropdown data:', event.fullData);
}
```

The event object contains:
- `nativeEvent`: The native DOM event
- `changedDropdown`: The dropdown node that changed
- `fullData`: Complete array of all dropdown configurations

</Step>

<Step title="Add Component Template">

Add the component to your template:

```html app.component.html
<div class="container mt-5">
  <h2>Dependent Dropdown Example</h2>
  
  <ngx-depend-dropdown 
    [data]="data" 
    [config]="config" 
    (onChange)="change($event)">
  </ngx-depend-dropdown>
</div>
```

</Step>

<Step title="Add Styling (Optional)">

Add some CSS to make your dropdowns look better:

```css app.component.css

```

</Step>

</Steps>

## Complete Example

Here's the full working component:

<CodeGroup>

```typescript app.component.ts
import { Component, OnInit } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { map } from 'rxjs';
import { NgxDependDropdownService } from 'ngx-depend-dropdown';
import { DropdownConfig, DropdownData } from 'NgxDependDropdown';

@Component({
  selector: 'app-root',
  templateUrl: './app.component.html',
  styleUrls: ['./app.component.css']
})
export class AppComponent implements OnInit {
  data: DropdownData = [];
  config: DropdownConfig = {};

  constructor(
    private http: HttpClient,
    private dropdownService: NgxDependDropdownService

  ngOnInit(): void {
    this.config = {
      unavailableType: 'disabled',
      containerClass: ['dropdown-container', 'row'],
      nodeClass: ['form-select', 'mb-3']
    };
    this.setDropdownData();
  }

  setDropdownData() {
    this.data = [
      {
        name: 'user-selector',
        visibility: () => true,
        selectedValue: null,
        placeholder: 'Select a user',
        values: [],
        setValues: () => {
          return this.http
            .get('https://jsonplaceholder.typicode.com/users')
            .pipe(
              map((data: any) => {
                return data.map((item: any) => ({
                  name: item.name,
                  id: item.id
                }));
              })
            );
        }
      },
      {
        name: 'dependent-selector',
        visibility: () => {
          return !!this.dropdownService.getNodeValue('user-selector');
        },
        dependent: 'user-selector',
        selectedValue: null,
        placeholder: 'Select an option',
        values: [],
        setValues: () => {
          return this.http
            .get('https://jsonplaceholder.typicode.com/users')
            .pipe(
              map((data: any) => {
                return data.map((item: any) => ({
                  name: item.email,
                  id: item.id
                }));
              })
            );
        }
      },
      {
        name: 'final-selector',
        visibility: () => {
          return !!this.dropdownService.getNodeValue('dependent-selector');
        },

        placeholder: 'Select final option',
```

## How It Works

1. **First dropdown** loads immediately and fetches user data from the API
2. **Second dropdown** remains disabled until you select a user
3. **Third dropdown** appears only after the second selection, with required validation
4. When you change any parent dropdown, all dependent children reset automatically

## Key Concepts

<CardGroup cols={2}>
  <Card title="Visibility Function" icon="eye">
    Controls when a dropdown is shown. Return `true` to show, `false` to hide or disable.
  </Card>
  <Card title="Dependent Property" icon="link">
    Links dropdowns together. When parent changes, dependent dropdowns reset.
  </Card>
  <Card title="setValues Function" icon="database">
    Provides dropdown options. Can return Observable or array.
  </Card>
  <Card title="Service Methods" icon="gear">
    Use `getNodeValue()` to check parent selections in visibility rules.
  </Card>
</CardGroup>
