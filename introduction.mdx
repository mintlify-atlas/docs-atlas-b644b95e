# ngx-depend-dropdown

A powerful yet lightweight Angular library that enables you to create linked dependent dropdowns with cascade behavior. Perfect for forms that require hierarchical selection flows like country/state/city, category/subcategory, or any dependent data relationships.

<Note>
  Library is under active development. Please report any issues or suggestions on [GitHub](https://github.com/yasincelebi/ngx-depend-dropdown).
</Note>

## Why ngx-depend-dropdown?

Building dependent dropdowns from scratch involves complex state management, visibility logic, and data synchronization. This library handles all that complexity for you with a simple, declarative API.

## How It Works

Define your dropdown configuration with visibility rules and dependencies:

```typescript
const dropdownData: DropdownData = [
  {
    name: 'country',
    visibility: () => true,
    setValues: () => this.http.get('/api/countries'),
    placeholder: 'Select Country'
  },
  {
    name: 'city',
    visibility: () => !!this.dropdownService.getNodeValue('country'),
    dependent: 'country',
    setValues: () => this.http.get('/api/cities'),
    placeholder: 'Select City'
  }
];
```

The library automatically:
- Shows/hides dependent dropdowns based on visibility rules
- Resets dependent values when parent selections change
- Manages state synchronization across the dropdown chain
- Handles async data loading with observables

## What You Can Build

- **Location Selectors**: Country → State → City cascades
- **Category Browsers**: Department → Category → Subcategory hierarchies
- **Product Configurators**: Brand → Model → Variant selections
- **Organizational Hierarchies**: Company → Division → Department → Team
- **Date/Time Pickers**: Year → Month → Day custom selectors
- **Dynamic Forms**: Any form where field visibility depends on previous selections

## Browser Support

ngx-depend-dropdown works with all modern browsers and Angular 13+.
